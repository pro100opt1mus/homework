Что было изменено в проекте Mario_OOP:

1. Создан отдельный класс Map
- Добавлены файлы map.hpp и map.cpp.
- Из Game вынесена вся логика, связанная с картой:
  - хранение двумерного массива карты;
  - очистка карты;
  - вывод карты на экран;
  - вывод счёта;
  - отрисовка объектов на карте;
  - проверка координат внутри карты.

2. Game теперь использует Map
- В Game.hpp вместо std::vector<std::vector<char>> map теперь хранится объект Map map.
- Методы Game::clear_map(), Game::show_map(), Game::put_score_on_map(), Game::put_object_on_map() теперь просто вызывают соответствующие методы класса Map.
- Логика уровней, движения и игрового цикла осталась в Game.

3. Разделён BaseObject и MovingObject
- BaseObject теперь хранит только базовые данные объекта:
  - координаты;
  - размеры;
  - тип объекта;
  - проверку столкновений;
  - базовые get/set методы.
- Создан MovingObject, который наследуется от BaseObject.
- В MovingObject перенесены:
  - вертикальная скорость;
  - горизонтальная скорость;
  - состояние полёта;
  - vert_move();
  - horiz_move();
  - set_vert_speed();
  - set_horiz_speed().

4. Статические и движущиеся объекты разделены по наследованию
- Brick наследуется от BaseObject, потому что он статический.
- MovingItem наследуется от MovingObject, потому что он движется.
- Mario работает через движущуюся иерархию объектов.

5. Исправлены места, которые ломались после разделения классов
- create_moving() теперь возвращает MovingObject*, а не BaseObject*.
- Убрано обращение к vertSpeed через BaseObject.
- Вместо прямого доступа к vertSpeed используется set_vert_speed().
- Методы vert_move() и horiz_move() теперь принадлежат MovingObject.

6. Разбиты длинные строки
- Длинные конструкторы, сигнатуры функций, условия и вызовы функций разбиты на несколько строк.
- Это сделано в Game.hpp, Game.cpp, BaseObject.hpp, BaseObject.cpp, Brick.hpp, Mario.hpp, Mario.cpp, MovingItems.hpp, map.hpp и map.cpp
